## Primary Key e Foreign Key

Quando encontramos uma coluna que possui um **ID**, como por exemplo `staff_id`, podemos dizer que ela é a **Primary Key** da tabela.

Podemos usar o **data value** dessa coluna para **relacionar** a tabela com outras. Em outras palavras, utilizamos esse valor para criar uma **relação entre tabelas**.

Normalmente, a **Primary Key** é a primeira coluna da tabela. Porém, essa mesma tabela pode ter outras chaves que fazem referência a tabelas externas. Esse tipo de chave é chamado de **Foreign Key** (*chave estrangeira*).
